# 1. ТРЕБОВАНИЯ К ПРОГРАММЕ

Программа должна реализовывать алгоритм Флойда-Уоршалла для поиска минимального пути в ориентированном графе, а также интерфейс к нему, который должен визуализировать построение графа кратчайших путей.

## 1.1. Исходные требования к программе

### 1.1.1. Требование к вводу исходных данных
* Ввод через интерфейс таблицу с матрицей смежности.
* Ввод через текстовый файл – матрицей смежности.
* Добавление/удаление вершин кнопками в интерфейсе.
* Генерация матрицы смежности для входных данных:
  1. кол-во вершин, степень вершины в пределах от 1 до N-1, веса.

### 1.1.2. Требования к визуализации

**Основные элементы интерфейса:**
* Холсты для отображения графа:
  * Холст с вводимым графом
  * Холст с текущим графом кратчайших путей

**Взаимодействие с холстом при помощи мыши:**
* Перемещение по холсту (прокрутка).
* Выделение элементов на холсте. Также выделяются в таблицах соответствующие им элементы.

**Панель управления с кнопками взаимодействия:**
* Кнопки управления шагами.
* Кнопки ввода/вывода из/в файл.
* Кнопки добавления/удаления вершин.
* Поле для отображения информации о текущем шаге алгоритма.

**Панель с логами программы:**
* Логирование действий пользователя (см. 1.1.2.1.).

**Функционал интерфейса:**
* На каждом шаге желтым цветом будут подсвечиваться данные в матрицах (соответствующие индексам вершин и их весам ребер между) и связанные с ними вершины и грани на холстах голубым цветом.
* Вывод номера шага, вывод итерации циклов - значения “k i j” алгоритма, номера вершин и веса ребер (описание).
* На холсте графа кратчайших путей отображать только возможные пути (!= inf).
* Автоматическая синхронизация холстов с таблицами при изменении.
* Функционал перемещения между шагами алгоритма.
* По умолчанию для новых вершин рёбра отсутствуют (inf).

#### 1.1.2.1. **Пример доступных действий:**
* Шаг назад из `k_i_j - a1_w1, b1_w1` в `k_i_j a0_w0, b0_w0`
* Шаг вперед из `k_i_j - a1_w1, b1_w1` в `k_i_j a2_w2, b2_w2`
* Пауза на `k_i_j - a1_w1, b1_w1`
* Установлена скорость `kx`
* Ввод из файла `filename.csv`
* Сброс
* Шаг `+kx` - произведен шаг из `N` в `N+kx`
* Добавление/удаление выделенной вершины.
* Выделение вершины/грани.
* Сохранение конечной матрицы смежности в `filename1.csv`, логов в `filename2.txt`

### 1.1.3. Требование к выходным данным
* Вывод через текстовый файл – матрицей минимальных расстояний:
  * Квадратная матрица N на N в формате CSV.
  * Формат CSV – описан в стандартах RFC 7111, RFC 4180.
* Логи программы:
  * Действия пользователя.
  * Подробнее см 1.1.2.1.
* Таблица с матрицей минимальных расстояний.
* Визуализация текущих данных алгоритма в виде графа.
* Выделение текущих элементов голубым.

### 1.1.4. Схематическое представление графического интерфейса Main:

| Холст 1 | Холст 2 |
| :--- | :--- |
| **Таблица 1** | **Таблица 2** |
| **Панель управления** | **Описание Шага** |
| Вывод логов построчно | Вывод логов построчно |

### 1.1.5. Схематическое представление расположения кнопок на панели управления:

| Шаг назад | Пуск/Пауза | Шаг вперед | Шаг N | Добавить вершину |
| :--- | :--- | :--- | :--- | :--- |
| **Сброс** | **Ввод из файла** | **Сохранение** | **Скорость kx** | **Удалить вершину** |

**Описание элементов:**
* **Холст 1** – вводимый граф (из таблицы 1).
* **Холст 2** – текущий граф алгоритма (из таблицы 2).
* **Таблица 1** – вводимая матрица смежностей.
* **Таблица 2** – текущая матрица алгоритма.
* **Панель управления** – включение раскраски и управления алгоритмом (шаг, сброс, перемотка).
* **Описание** – описание текущего шага алгоритма.
